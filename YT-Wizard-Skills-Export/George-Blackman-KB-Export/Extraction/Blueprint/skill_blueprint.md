# George Blackman Skill Blueprint

## Purpose

This blueprint turns the extracted George Blackman channel intelligence into a practical skill roadmap.

It is based on the AI layer only.

The goal is not to make one giant writing skill.

The goal is to build a stack of focused skills that work together:
- extract
- analyze
- detect
- convert
- write

## Core Insight

The strongest reusable ideas in this channel are not random tips.

They are systems:
- setup -> tension -> payoff
- target -> transformation -> stakes
- retention graph review loop
- segment-first drafting
- AI collaboration loop
- visual orientation loop

So the best skills to build are the ones that can:
1. identify these systems
2. apply them
3. convert them into repeatable workflows

## Recommended Build Order

1. `channel-best-practices-extractor`
2. `video-script-analyzer`
3. `framework-detector`
4. `analysis-to-writer-skill-builder`
5. `ai-script-collaboration-helper`

## Why This Order

### 1. `channel-best-practices-extractor`

This creates the intelligence layer.

Without it, the later skills have less reliable source material.

### 2. `video-script-analyzer`

This is the next highest-leverage skill because it lets you study any strong video, not just teaching channels.

That makes it useful immediately for client work, research, and adaptation.

### 3. `framework-detector`

Once you can analyze a script, the next useful capability is naming and classifying the framework inside it.

This becomes the bridge between analysis and recreation.

### 4. `analysis-to-writer-skill-builder`

This turns analyzed frameworks into actual writing skill specs.

This is where channel intelligence and single-video intelligence start becoming reusable creative assets.

### 5. `ai-script-collaboration-helper`

This is valuable, but more of an optimizer than a foundation.

It should come after the higher-level thinking skills are locked.

## Skill 1: Channel Best Practices Extractor

### Job

Read one scriptwriting-focused teaching channel and extract:
- best practices
- writing rules
- named frameworks
- inferred frameworks
- anti-patterns
- worldview
- canonical phrases
- process workflows
- skill opportunities

### Input

- one channel folder
- transcript markdown files already present
- optional spreadsheet support file

### Output

- Human per-video files
- AI per-video files
- Human channel files
- AI channel files
- blueprint scaffold

### Why It Matters

This creates the source material for almost everything else.

### Readiness

Ready now.

## Skill 2: Video Script Analyzer

### Job

Analyze one strong video script on any topic and explain:
- why it works
- what structure it uses
- where tension and payoff are created
- how curiosity is managed
- how stakes are introduced
- how pacing and transitions are handled
- what parts are universal vs topic-specific
- how to recreate the underlying script logic in another topic

### Input

- one video transcript or finished script
- optional target topic for adaptation

### Output

- script structure breakdown
- retention logic breakdown
- framework candidates
- recreation notes
- adaptation guidance

### Why It Matters

This is the bridge from admiration to replication.

It lets you study successful videos outside scriptwriting-teaching channels.

### Readiness

Should be built next.

## Skill 3: Framework Detector

### Job

Identify whether a script uses:
- setup -> tension -> payoff
- target -> transformation -> stakes
- character -> concept -> stakes
- link -> curiosity -> promise
- visual orientation loop
- other recurring inferred frameworks

It should also detect:
- named frameworks
- unnamed but repeated patterns
- framework quality
- where the framework breaks down

### Input

- one script
- one transcript
- or analyzer output

### Output

- detected frameworks
- inferred frameworks
- confidence labels
- stage mapping
- evidence snippets

### Why It Matters

This is the classification layer.

It makes later recreation and skill building much easier.

### Readiness

Strong third build.

## Skill 4: Analysis to Writer Skill Builder

### Job

Take the output of the analyzer and framework detector and turn it into:
- a writer skill concept
- a reusable workflow
- writing rules
- prompt scaffolding
- output expectations

This skill should answer:
- what kind of writing skill can be built from this analysis?
- what should the skill do?
- what should it not do?
- what input should it take?
- what output should it produce?

### Input

- analyzer result
- framework detection result
- optional channel intelligence

### Output

- skill spec
- workflow spec
- rule set
- recommended prompt architecture

### Why It Matters

This is the conversion layer from research to assets.

### Readiness

Build after analyzer and framework detector.

## Skill 5: AI Script Collaboration Helper

### Job

Help write scripts with AI in a structured way:
- brainstorm collaboratively
- isolate one task at a time
- refine segment by segment
- compare AI draft vs final draft
- update style guide and training docs

### Input

- topic
- audience
- structure
- current script draft
- style guide

### Output

- collaborative drafting workflow
- revision suggestions
- learning report
- updated style/training notes

### Why It Matters

This operationalizes George's AI workflow.

### Readiness

Useful after the thinking skills are stable.

## Shared Design Rules Across Future Skills

- Keep skills focused, not bloated
- Preserve confidence labels
- Preserve explicit vs inferred distinctions
- Preserve named vs inferred framework distinctions
- Keep universal vs creator-specific advice separate
- Prefer structured output over vague commentary
- Optimize for real application, not just summary
- Keep human review possible at every stage

## Skills That Can Wait

- multi-channel best-practices synthesizer
- niche-specific writer skills
- platform-specific writer variants
- automation-heavy orchestration

These are better once the first five skills are tested.

## Future Multi-Channel Direction

Later, build a multi-channel layer that can:
- compare channels
- connect overlapping ideas
- identify consensus frameworks
- identify creator-specific strengths
- detect who is best at teaching what
- build "best of multiple channels" skill opportunities

That should come after the one-channel and single-video systems are proven.
