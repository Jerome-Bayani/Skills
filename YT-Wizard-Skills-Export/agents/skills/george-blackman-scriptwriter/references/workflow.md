# Workflow Reference

This skill follows a George Blackman-inspired system.

## Core Sequence

1. Clarify audience and framing
2. Decide payoffs first
3. Write setups that make the payoffs matter
4. Build tension without collapsing curiosity
5. Write the hook after the body is clear
6. Add the CTA
7. Run a final self-review

## Why It Works

- It reduces blank-page overwhelm
- It locks structure before prose expands
- It keeps retention in mind during writing, not just after publishing
- It uses AI as support, not as a replacement thinker

## Structural Center

### Setup -> Tension -> Payoff

The setup should open anticipation.

The tension should:
- mirror the viewer's current problem or wrong move
- show why the current approach fails
- move toward the answer without giving it away too early

The payoff should:
- resolve the segment clearly
- feel worth the wait
- connect cleanly to what comes next

## Hook Logic

### Target -> Transformation -> Stakes

The hook should identify:
- who this is for
- what they will get
- what it costs them to ignore it

Avoid:
- excessive detail
- stacked credibility claims
- non-critical information
- decorative metaphor before the real promise

## CTA Logic

### Link -> Curiosity -> Promise

The CTA should feel like:
- continuation
- deeper value
- a clear reason to click

Not:
- a generic ask

## Research Rule

Use a research pass when the script depends on:
- facts
- statistics
- history
- markets
- geography
- named examples needing verification

## Direct Generator Pattern

This skill is direct by default.

But it should still use checkpoints where the cost of being wrong is highest:
- title and hook direction
- payoff structure
